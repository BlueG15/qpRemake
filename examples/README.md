# Examples

This folder contains small TypeScript examples that demonstrate common usage patterns.

1. `01-basic-setup.ts` – create a system, load players, start the game.
2. `02-custom-settings.ts` – customize `DefaultSetting`.
3. `03-custom-renderer.ts` – implement the `qpRenderer` interface.
4. `04-process-single-action.ts` – create and process a single action.
5. `05-save-and-load.ts` – serialize and restore game state.
6. `06-auto-input.ts` – use automatic input handling modes.
7. `07-read-game-state.ts` – inspect zones and localized system data.
8. `08-add-gamerule.ts` – add an additional game rule.
9. `09-modding-api.ts` – extend runtime registries with `ModdingAPI`.
10. `10-run-test-suite.ts` – run the built-in test suite helpers.

> These are reference snippets intended for onboarding and integration guidance.
